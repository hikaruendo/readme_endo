\begin{comment}
まず，環境光がKinectの赤外線深度センサ情報の取得に与える影響について先に
述べた3つの条件でSLAMを行った結果を図\ref{環境地図(条件1)}から図\ref{環境地図(条件3)}に示す．条件1(図\ref{環境地図(条件1)})と条件2(図\ref{環境地図(条件2)})では，環境地図を構築をすることは可能であるが，条件3(図\ref{環境地図(条件3)})では，
環境地図を構築することができなかった．図\ref{環境地図(条件3)}に示したのはオドメト
リが計算できず，マッピングが続けられない場合に表示されるエラー画面である．
このようにオドメトリが失われる原因としては，以下の4つが考えられる．
\begin{description}
\setlength{\parskip}{0cm}
\setlength{\itemsep}{0cm}
\item[I]移動速度が速い
\item[I\hspace{-.1em}I]十分な特徴量(輝度)が得られない
\item[I\hspace{-.1em}I\hspace{-.1em}I]深度情報と色情報のミスアライメント
\item[I\hspace{-.1em}V]深度測定範囲内に物体が無い
\end{description}
条件3(図\ref{環境地図(条件3)})では，暗い領域にある対象物体から画像を取得
する実験を行ったために，特徴量の検出が十分に行われなかったことがエラーの原
因であると考えられる．このことから実験時に対象物体にはある程度の環境光が
当たることが必要であるといえる．また条件1(図\ref{環境地図(条件1)})では，環境地図の構築は行われたが，特徴点
の重ね合わせが正しく行われていない．画像取得対象物体に直射日光が
強く当たると，これに含まれる赤外線の反射を赤外線深度センサが取得
すると考えられる．そのため正しい特徴量を十分に取得できず，少ない特
徴量から不適切な重ね合わせが行われ，その誤差が蓄積したことが正しい環境地
図の得られない原因であると考える．
これらの結果から条件2を実験環境として採用した.
